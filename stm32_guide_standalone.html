Terminal Setup

First, let's set up your terminal environment for development.

Open Terminal

Press Cmd + Space, type "Terminal", and press Enter.

Check Your Shell

Verify you're using zsh (the default on macOS):

echo $SHELL

You should see: /bin/zsh

Install Homebrew

If you don't have Homebrew installed, install it:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Update PATH

Add Homebrew to your PATH if needed:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
source ~/.zshrc

PlatformIO Installation

PlatformIO is a cross-platform IDE for embedded development that makes STM32 programming much easier.

Install Python

PlatformIO requires Python 3.6+:

brew install python3

Install PlatformIO Core

Install the command-line version:

pip3 install platformio

Verify Installation

Check that PlatformIO is installed correctly:

pio --version

You should see version information displayed.

Success! PlatformIO is now installed and ready to use.

Creating Your First Project

Let's create a simple LED blink project for the STM32 Blue Pill.

Initialize Project

Create a new project directory and initialize it:

mkdir stm32_blink
cd stm32_blink
pio project init --board bluepill_f103c8

Create Main Code

Create the main source file:

mkdir -p src
cat > src/main.cpp << 'EOF'
#include 

#define LED_PIN PC13

void setup() {
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_PIN, HIGH);
  delay(1000);
  digitalWrite(LED_PIN, LOW);
  delay(1000);
}
EOF

Build the Project

Compile your code:

pio run

This will download necessary toolchains and libraries, then compile your code.

Upload & Run

Now let's upload the code to your STM32 Blue Pill.

Connect Your Device

Connect your STM32 Blue Pill to your Mac via USB-to-Serial adapter.

Typical STM32 Blue Pill to USB-Serial connection

Set Boot Mode

Set BOOT0 jumper to position 1 (bootloader mode).

Find Serial Port

Identify your serial port:

ls /dev/tty.usbserial*

Note the port name (e.g., /dev/tty.usbserial-1420).

Upload Code

Upload your compiled code:

pio run --target upload

Reset to Run

Set BOOT0 jumper back to position 0 and press the reset button. Your LED should start blinking!

How-To Guides

Fix Serial Port Issues

Common solutions for serial port connection problems.

Device Not Found

Problem: No /dev/tty.usbserial* devices found

Solution:

  1. Check USB cable connection
  2. Install CH340 or CP2102 drivers
  3. Try a different USB port
  4. Restart your Mac

Permission Denied

Problem: Can't access serial port

Solution:

sudo dseditgroup -o edit -a $USER -t user dialout

Then log out and back in.

Upload Fails

Problem: Upload times out or fails

Solution:

  1. Ensure BOOT0 is set to 1
  2. Press reset before upload
  3. Check baud rate (9600 or 115200)
  4. Try different upload protocol

Check Device Connection

Verify your STM32 Blue Pill is properly connected and recognized.

List USB Devices

system_profiler SPUSBDataType

Look for your USB-to-Serial adapter in the output.

Check Serial Ports

ls -la /dev/tty.*

Look for devices like tty.usbserial-* or tty.wchusbserial*.

Test Connection

pio device list

This shows all available serial devices that PlatformIO can use.

Upload with ST-Link

Use ST-Link V2 programmer for more reliable uploads and debugging.

Install ST-Link Tools

brew install stlink

Connect ST-Link

Wire ST-Link to your Blue Pill:

ST-LinkBlue Pill
3.3V3.3V
GNDGND
SWDIODIO
SWCLKDCLK

Configure platformio.ini

[env:bluepill_f103c8]
platform = ststm32
board = bluepill_f103c8
framework = arduino
upload_protocol = stlink
debug_tool = stlink

Upload

pio run --target upload

Debug with GDB

Set up hardware debugging with GDB and ST-Link.

Install GDB

brew install --cask gcc-arm-embedded

Add Debug Configuration

Add to your platformio.ini:

debug_tool = stlink
debug_init_break = tbreak setup
build_type = debug
build_flags = -DDEBUG -O0 -g3 -ggdb

Start Debug Session

pio debug

Manage Libraries

Install and manage libraries for your STM32 projects.

Search Libraries

pio lib search "sensor name"

Install Library

pio lib install "library-name"

List Installed Libraries

pio lib list

platformio.ini Configuration

Complete reference for STM32 project configuration.

[env:bluepill_f103c8]
platform = ststm32
board = bluepill_f103c8
framework = arduino

; Upload settings
upload_protocol = serial
upload_port = /dev/tty.usbserial-1420
upload_speed = 115200

; Monitor settings
monitor_port = /dev/tty.usbserial-1420
monitor_speed = 9600

; Build settings
build_flags = 
    -DBOARD_NAME="BluePill F103C8"
    -DSERIAL_TX_BUFFER_SIZE=1024
    -DSERIAL_RX_BUFFER_SIZE=1024

; Library dependencies
lib_deps = 
    arduino-libraries/Servo@^1.1.8
    adafruit/Adafruit Sensor@^1.1.4

Explanation

Why PlatformIO?

Understanding the benefits of PlatformIO over traditional IDEs.

PlatformIO Advantages:
  • Cross-platform: Works on macOS, Windows, and Linux
  • Unified interface: Same commands for all platforms
  • Library management: Easy dependency management
  • Multiple frameworks: Arduino, STM32Cube, Mbed, etc.
  • Built-in debugger: Hardware debugging support

Traditional STM32 development required separate tools for each platform, complex toolchain setup, and manual library management. PlatformIO simplifies this entire process into a single, unified development environment.

STM32 Bootloader

How the STM32 bootloader works and why boot modes matter.

The STM32F103C8T6 has a built-in bootloader in system memory that allows programming via UART, I2C, or SPI without external programmers. This bootloader checks the BOOT0 and BOOT1 pins at startup to determine the boot source.

STM32 boot process flow based on BOOT0 pin state

Boot Sequence

  1. STM32 powers up and reads BOOT0/BOOT1 pins
  2. Based on pin states, jumps to appropriate memory location
  3. If in bootloader mode, waits for programming commands
  4. If in flash mode, executes user application

Cortex-M Architecture

Understanding the ARM Cortex-M3 architecture in STM32F103C8T6.

The STM32F103C8T6 uses the ARM Cortex-M3 core, a 32-bit RISC processor designed for microcontroller applications. It features:

  • 32-bit ARM architecture: Efficient instruction set
  • 72MHz maximum frequency: Fast processing for real-time applications
  • 64KB Flash memory: Program storage
  • 20KB SRAM: Runtime memory
  • 37 GPIO pins: Digital I/O with various functions
  • 3x USART: Serial communication
  • 2x I2C: Inter-integrated circuit bus
  • 2x SPI: Serial peripheral interface
Memory Map: The Cortex-M3 uses a unified memory map where Flash, SRAM, and peripherals are all memory-mapped, making programming more straightforward.

macOS Development Tips

Platform-specific tips and tricks for STM32 development on macOS.

🍎 Terminal Shortcuts

  • ⌘+T - New terminal tab
  • ⌘+D - Split terminal pane
  • ⌘+K - Clear terminal
  • ⌃+C - Stop running process

🔧 System Preferences

  • Allow apps from "Anywhere" in Security
  • Enable "Developer Mode" for debugging
  • Configure USB device access
  • Set up proper file permissions

⚡ Performance Tips

  • Use SSDs for faster compilation
  • Close unnecessary applications
  • Enable compiler caching
  • Use multiple cores for builds

🛡️ Security Notes

  • Gatekeeper may block unsigned tools
  • Use xattr -d com.apple.quarantine
  • System Integrity Protection (SIP) affects drivers
  • Some USB drivers require kernel extensions
Apple Silicon (M1/M2) Users: Some USB-to-serial adapters may have compatibility issues. Try using Rosetta 2 for problematic tools: arch -x86_64 pio run
